Introduction

Welcome to the CSS and JavaScript technical documentation page! This guide provides an in-depth exploration of key concepts, syntax, and practical examples to help you master both CSS and JavaScript. We'll cover everything from basic to intermediate JavaScript and from basic to advanced CSS, with detailed explanations for each concept.

Introduction to CSS

What is CSS?

CSS (Cascading Style Sheets) is a stylesheet language used to describe the presentation of a document written in HTML.

It controls the layout, colors, fonts, and overall design of web pages, making them visually appealing and responsive.

Why Use CSS?

  • Separation of Concerns: Keeps styling separate from content, making code easier to maintain.
  • Reusability: Styles can be reused across multiple pages.
  • Consistency: Ensures a uniform look and feel across a website.

Basic Syntax

CSS rules consist of selectors and declarations:

selector {
property: value;
}
  • Selector: Targets the HTML element(s) to style.
  • Property: The style attribute to modify (e.g., color, font-size).
  • Value: The value assigned to the property.
h1 {
color: blue; /* Changes text color to blue */
font-size: 24px; /* Sets font size to 24 pixels */
}

Selectors

Selectors determine which elements the CSS rules apply to. There are several types of selectors:

  1. Element Selector

    • Targets HTML tags directly.
    • Doesn't start or end with a special character.
    p {
    color: gray;
    }

  2. Class Selector

    • Targets elements with a specific class.
    • The class name is preceeded by a dot .
    .highlight {
    background-color: yellow;
    }

  3. ID Selector

    • Targets a unique element by its ID.
    • The ID is preceeded by a hashtag #
    #header {
    font-weight: bold;
    }

  4. Attribute Selector

    • Targets elements based on their attributes.
    • Tags are followed by a brackets and the attribute within (e.g, input[type="text"] )
    input[type="text"] {
    border: 1px solid black;
    }

  5. Universal Selector

    • Targets all elements.
    • Represented by a star *.
    * {
    margin: 0;
    padding: 0;
    }

Box Model

Every HTML element is represented as a rectangular box, consisting of:

  1. Content: The inner text or media.
  2. Padding: Space between the content and the border.
  3. Border: The outline around the padding.
  4. Margin: Space outside the border.
div {
width: 200px;
padding: 10px;
border: 2px solid black;
margin: 20px;
}

Key Points:

  • Width and Height: Include padding and border unless using box-sizing: border-box.
    • * {
    box-sizing: border-box;
    }
  • Margin Collapse: Adjacent vertical margins collapse into a single margin.

Flexbox and Grid

Modern layout techniques include:

  • Flexbox: A one-dimensional layout model for arranging items in rows or columns.
  • Grid: A two-dimensional layout system for designing complex layouts.

Flexbox

Main Concepts

  • Container: The parent element that holds flex items.
  • Items: The child elements inside the container.
  • Axes: Main axis (horizontal or vertical) and cross axis.
.container {
display: flex;
justify-content: center; /* Centers items horizontally */
align-items: center; /* Centers items vertically */
}

Grid

Main Concepts

  • Grid Container: The parent element.
  • Grid Items: The child elements.
  • Rows and Columns: Define the grid structure.
.grid-container { display: grid;
grid-template-columns: repeat(3, 1fr);
gap: 20px;
}

Intermediate CSS

Pseudo-Classes and Pseudo-Elements

Pseudo-classes and pseudo-elements allow you to style specific states or parts of elements:

  1. Pseudo-Classes

    • Target elements based on their state (e.g., :hover, nth-child).

    • Example:

    a:hover {
    color: red; /* Changes link color on hover */
    }

  2. Pseudo-Elements

    • Style specific parts of an element (e.g., ::before, ::after).

    • Example:

    p::first-line {
    font-weight: bold; /* Styles the first line of a paragraph */
    }

Transitions and Animations

CSS transitions and animations add dynamic effects to elements:

  1. Transitions

    • Smoothly change property values over time.

    • Example:

    button {
    background-color: blue;
    transition: background-color 0.5s ease;
    }
    button:hover {
    background-color: green;
    }

  2. Animations

    • Define keyframes for more complex animations.

    • Example:

    3. @keyframes slideIn { from { transform: translateX(-100%); }
    to { transform: translateX(0); }
    }
    .box {
    animation: slideIn 2s ease-in-out;
    }

Responsive Design with Media Queries

Media queries allow you to apply styles based on screen size or device type:

@media (max-width: 768px) {
body {
font-size: 14px;
}
}

Key Points:

  • Breakpoints: Common breakpoints include 320px, 480px, 768px, 1024px, and 1200px.
  • Mobile-First Design: Start with styles for small screens and use min-width to add styles for larger screens.

Advanced CSS

CSS Variables

CSS variables (custom properties) allow you to define reusable values:

:root {
--primary-color: #3498db;
}
button {
background-color: var(--primary-color);
}

Key Points:

  • Scope: Variables defined in :root are global.
  • Fallbacks: Provide a fallback value if the variable is not defined.
color: var(--text-color, black);

Advanced Layout Techniques

Advanced techniques like CSS Grid and positioning enable precise control over layouts:

  1. Positioning

    • Static: Default positioning.
    • Relative: Positioned relative to its normal position.
    • Absolute: Positioned relative to the nearest positioned ancestor.
    • Fixed: Positioned relative to the viewport.
    • Sticky: Toggles between relative and fixed positioning.
    2. .sticky-header {
    position: sticky;
    top: 0;
    }

  2. Grid Layouts

    Use grid-template-areas for named areas.

    .container {
    display: grid;
    grid-template-areas:
    "header header"
    "sidebar main"
    "footer footer";
    }
    .header {
    grid-area: header;
    }

Accessibility and Best Practices

Follow accessibility guidelines to ensure your designs are inclusive:

  • Semantic HTML: Use proper HTML tags (e.g., <header>, <main>).
  • Contrast: Ensure sufficient contrast between text and background.
  • ARIA Roles: Use ARIA roles for dynamic content.
<button aria-label="Close">X</button>

Introduction to javascript

What is JavaScript?

JavaScript is a programming language that adds interactivity and dynamic behavior to web pages.

It enables features like animations, form validation, and data manipulation.

Why Use JavaScript?

  • Interactivity: Makes websites dynamic and engaging.
  • Versatility: Works on both client-side and server-side (Node.js).
  • Ecosystem: Extensive libraries and frameworks (e.g., React, Vue, Angular).

Basic Syntax

JavaScript code is executed by the browser and follows a simple structure:

// Single-line comment
/* Multi-line comment */
console.log("Hello, World!"); // Outputs text to the console

Variables and Data Types

Variables store data values. JavaScript supports several data types:

  • Strings: Text (e.g., "Hello").
  • Numbers: Integers or decimals (e.g., 42, 3.14).
  • Booleans: True or false values (e.g., true, false).
let name = "John"; // String
let age = 25; // Number
 let isStudent = true; // Boolean

Functions

Functions are reusable blocks of code that perform specific tasks:

function greet(name) {
return "Hello, " + name + "!";
}
console.log(greet("Alice")); // Output: Hello, Alice!

DOM Manipulation

The Document Object Model (DOM) represents the structure of a web page.

JavaScript can manipulate the DOM to dynamically update content.

document.getElementById("demo").innerHTML = "Updated Content!";

Intermediate JavaScript

Arrays and Objects

Arrays and objects are used to store collections of data:

  • Arrays: Ordered lists of values.
  • Objects: Key-value pairs.
let fruits = ["apple", "banana", "cherry"];
let person = { name: "John", age: 25 };

Loops and Iteration

Loops allow you to iterate over arrays or perform repetitive tasks:

for (let i = 0; i < fruits.length; i++) {
console.log(fruits[i]);
}

Event Handling

Event listeners allow you to respond to user interactions:

button.addEventListener("click", function() {
alert("Button clicked!");
});

Error Handling

Use try...catch to handle errors gracefully:

try {
throw new Error("Something went wrong!");
} catch (error) { console.error(error.message);
}
